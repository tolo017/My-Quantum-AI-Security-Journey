# Day 10: Web Vulnerabilities II

Today we move beyond basic injections and look at how logic and access control break.

## 1. IDOR (Insecure Direct Object Reference)
- **Concept:** Changing a number in a URL to see someone else's data.
- **Example:** You log in and see `shop.com/profile?id=100`. You change it to `id=101`. If you see *another* user's profile, that is **IDOR**.

## 2. LFI/RFI (Local/Remote File Inclusion)
- **Concept:** Tricking the website into showing you files from the server's own disk.
- **Example:** `site.com/view?page=contact.php`. You change it to `page=/etc/passwd`.

## 3. CSRF (Cross-Site Request Forgery)
- **Concept:** Tricking a logged-in user into clicking a link that performs an action (like changing their password) without them knowing.
